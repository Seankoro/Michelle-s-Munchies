# Menu spec — Michelle's Munchies

Draft for review. This replaces the demo catalogue with a real menu. Prices are sensible
Singapore home bakery starting points that Michelle adjusts in admin later. Sizes are metric.
Nothing is built or loaded until this is signed off.

## Sections

Customer friendly headers, six in total. I added Brownies and bars as a sixth, because brownies
are a staple and did not sit well inside the other five. Say the word if you would rather fold them
into another section.

1. Cakes and cheesecakes
2. Cookies and macarons
3. Brownies and bars
4. Puddings and custards
5. Confections
6. Breads and bakes

## Dietary handling

- The menu filter stays the main tool. It filters on the fixed tags set per item.
- The line appears in two places. A short version sits beside the dietary filter on the menu page.
  Draft short wording. "Many treats can be made to suit dietary needs. Filter to browse what already
  fits, or add a note with your order." A fuller version goes on the FAQ and contact page, where there
  is room to explain that many treats can be adapted, give examples like a no pork beef gelatine
  version or a crustless gluten free cheesecake, and invite a note with the order.
- Items with a real made to order swap state it in their own description. These are the jelly
  cheesecake and panna cotta for a no pork beef gelatine version.
- Gelatine is pork by default, so the jelly cheesecake and panna cotta are not tagged vegetarian or
  no pork. The agar fruit jelly is the fully vegan and no pork option.

## Admin flavour editor, the feature to build

Today the admin product form cannot edit options at all. It shows a placeholder. We build an inline
editor inside the product modal so Michelle manages this herself.

- Add and remove option groups, each with a name and a required switch.
- Add and remove values inside a group, each with a label, a price change in dollars, and an
  availability switch.
- The availability switch is the per flavour sold out control. The storefront already greys out and
  blocks a value whose availability is off, so this closes the loop.
- Wire the editor through the existing createProductAction and updateProductAction so it saves to
  Postgres with the rest of the product.
- Small correctness fix at the same time. Do not pre select a sold out value, and block adding a sold
  out value to the cart, so a flavour that is off cannot slip through.

## Build approach

1. Build the admin flavour editor and the small sold out fixes.
2. Load all items above with their options into the database, replacing the demo products.
3. Add the bundles and build a box templates.
4. Set the homepage best seller and recommended flags as marked.
5. Write each item a short and a full description in plain copy, metric sizes, with the made to order
   line on the few adaptable items.
